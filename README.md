# 天气预报插件

## 功能

- 定时向指定群聊推送天气播报（可由 LLM 生成个性化内容）
- 每个群聊可配置不同推送地点
- `/天气 地点` 或 `/weather 地点` 手动查询实时天气及未来预报
- LLM 天气查询工具，麦麦被问到天气相关问题时可自动调用

## 快速开始

1. 在 [高德开放平台](https://console.amap.com/dev/key/app) 申请 Web服务 类型的 API Key
2. 在 WebUI 或 `config.toml` 中填入 Key 并配置群聊

## 配置

```toml
[plugin]
enabled = true

[weather]
amap_key = "your_amap_key_here"

[groups]
# 列表格式，每项为 "群号 地点"
target_groups = ["123456789 河北省", "987654321 北京市"]

[schedule]
broadcast_time = "08:00"

[broadcast]
max_length = 200

[tool]
enabled = true
default_days = 3
```

### 群聊配置格式

每项格式为 `群号 地点名称`，地点支持省、市、区级别：

- `772915232 河北省` — 推送河北省各市天气汇总
- `123456789 北京市` — 推送北京市天气
- `987654321 朝阳区` — 推送朝阳区天气

## 指令

| 指令 | 说明 |
|------|------|
| `/天气 地点` | 查询指定地点实时天气及未来3天预报 |
| `/weather 地点` | 同上（英文别名） |

## 注意事项

### 429 限流风险

定时播报时，每个群聊会独立请求天气数据并调用 LLM 生成播报。如果配置了大量群聊，短时间内会产生密集的 LLM 请求，**可能触发 429 限流**。

- 插件已在每个群播报之间加入 2 秒间隔
- 群聊数量较多时（>10个），建议关注日志是否出现 429
- 如遇限流，可减少群聊数量或联系模型服务商提升配额

### API 配额

高德地图个人开发者每日有免费调用额度，正常使用足够。大量群聊配置不同地区时注意配额消耗。
